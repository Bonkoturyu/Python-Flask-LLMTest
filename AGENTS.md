# Codex作業ガイドライン

- 既存コード全体を無目的に再読しない。必要なファイルと一次資料だけを調査する。
- 実装範囲を依頼内容に限定し、追加改善は実装せず候補として報告する。
- 長い説明やコード全文の再掲を避け、要点を簡潔に伝える。
- 同じ方法で2回失敗した場合は作業を続行せず、原因候補と必要な判断を報告する。
- SubAgentは必要性を説明できる場合のみ使用し、同時に1体までとする。
- 明確に分離できる調査・実装・テストは、上記の制限内でSubAgentへ委譲する。
- CodexからSubAgentへ委譲する場合は、可能ならTerraまたはLunaを使用する。
- 親は要件解釈・設計判断・統合・最終検証に集中する。
- 親はSubAgentの成果を全文再生成せず、差分と要点だけを確認する。
